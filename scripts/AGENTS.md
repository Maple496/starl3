# Scripts 文件夹操作规范

## 文件夹用途

`scripts` 文件夹**只能存放功能相关的 Python 代码**。

## 文件存放规则

| 类型 | 存放位置 |
|------|----------|
| 功能代码 | `scripts/` (当前文件夹) |
| 测试文件 | `../tests/` (上一级文件夹的 tests 目录) |
| 虚拟环境 | `../venv/` (上一级文件夹的 venv 目录) |

## Git 操作规范

- **Git 工作目录**: 上一级文件夹 (`../`)
- **提交范围**: 只提交 `scripts/` 文件夹内的内容
- **忽略项**: 
  - `test` 文件/文件夹
  - `venv` 文件夹
  - 其他非 scripts 的内容

## 目录结构示例

```
starl3/
├── .git/              # Git 仓库根目录
├── .gitignore         # 在根目录配置忽略规则
├── scripts/           # ✅ 本文件夹：只放功能代码
│   ├── AGENTS.md      # 本文件
│   ├── main.py
│   └── ...
├── tests/             # ✅ 测试文件放这里
│   └── test_xxx.py
└── venv/              # ✅ 虚拟环境放这里
    └── ...
```
